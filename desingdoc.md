主游戏循环
游戏有两个核心状态：CLIMBING（攀爬） ↔ SHOP（商店）

商店 → 选择升级/装备 → 点击"再次攀爬"
  ↓
攀爬阶段:
  ① 按住鼠标左键 → 判定条出现，指针来回弹跳
  ② 松开左键 → 判定成功/失败
  ③ 成功 → 推石上山 + 消耗体力 + 检查检查点奖励
  ④ 失败 → 不推动，不消耗体力，但开始滑落计时
  ⑤ 若长时间不操作 → 冒汗(1s) → 不可逆滑落(3s) → 跌回山底
  ⑥ 体力耗尽 → 判定区消失，强制最大速度滑落
  ↓
巨石滑到高度0 → 结算货币 → 返回商店

每帧更新顺序：体力恢复 → 判定条移动 → 体力检查 → 滑落更新 → 推动动画 → 结束检查 → 摄像机 → 检查点通知 → HUD → 渲染

核心机制拆解
1. 判定条（Judgment Bar）
参数	值
判定条宽度	300px
判定条高度	24px
指针速度	200 px/s
成功区最大比例	60%（满体力时）
成功区最小宽度	5px（低于此消失）
机制：成功区宽度 = 条总宽 × (当前体力/最大体力) × 成功区比例。体力越低，判定窗口越小。成功区位置每次随机偏移。

2. 体力系统（Stamina）
参数	值
基础最大体力	100
每次成功消耗	10
基础恢复速率	0.25/秒
机制：只有判定成功才扣体力，失败不扣。体力不直接显示，通过成功区宽度间接表现。体力耗尽→成功区消失→无法成功→强制滑落。

3. 滑落系统（Slide）
参数	值
冒汗延迟	1.0秒（不操作后）
冒汗滑落速度	10 单位/秒
不可逆滑落延迟	再等3.0秒
滑落初始速度	40 单位/秒
滑落最大速度	400 单位/秒
加速持续时间	2.0秒
三阶段：正常 → 冒汗（可恢复，缓慢下滑）→ 滑落（不可逆，加速到底）

4. 推动距离
参数	值
基础推动距离	40 游戏单位
动画时长	0.32秒
缓动函数	easeOutCubic

总结
这是一个以希腊神话西西弗斯为主题的增量/节奏类游戏。核心玩法是通过判定条的节奏操作推石上山，体力作为资源限制每次攀爬的高度，滑落机制制造紧迫感。玩家通过多次"推石→滑落→赚钱→升级"的循环逐步提升能力，解锁更高检查点和稀有货币，最终制作神器。所有数值集中在 config.ts 中管理。
