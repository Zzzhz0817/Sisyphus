# 四山系统开发规格书

**Four Mountains System — Dev Spec v1.0**
2026.03.19

---

## 1. 功能概述

将现有的单一无限山坡改为4座相连的独立山脉，每座山有独立的高度、坡度、视觉风格和推力惩罚。玩家依次攀爬，登顶后滑入下一座山的山脚，触发商店后继续。

---

## 2. 地形结构

整个世界是一条连续的地形线，从左到右由以下段落拼接：

```
[山1上坡] → [山1山顶] → [山1下坡] → [平地] → [山2上坡] → [山2山顶] → [山2下坡] → [平地] → ...
```

### 2.1 每座山的结构

每座山由三段组成：

| 段落 | 说明 |
|------|------|
| 上坡 | 玩家推石头的主要区域，坡度固定（每座山不同） |
| 山顶 | 短暂的平台/圆弧过渡，视觉上明确"到顶了" |
| 下坡 | 比上坡更陡的斜面，巨石自动滚下，玩家不操作 |

山与山之间有一段**短暂平地**过渡，然后接下一座山的上坡起点。

### 2.2 山脉参数

| 山脉 | 高度 | 坡度 | 推力系数 | 视觉风格 |
|------|------|------|---------|---------|
| 山1：塔耳塔洛斯丘陵 | 2,000 | 基准坡度 | ×1.0 | 基础色，较缓的斜面 |
| 山2：冥府斜坡 | 5,000 | 稍陡 | ×0.5 | 颜色更深，坡度更陡 |
| 山3：奥林匹斯山腰 | 12,000 | 更陡 | ×0.35（待定） | 更深色调，明显陡峭 |
| 山4：众神之巅 | 24,000 | 最陡 | ×0.25（待定） | 最深色调，最陡坡面 |

> **推力系数**：在该山坡上，实际推动距离 = basePushDistance × pushMultiplier × 推力系数。山2的×0.5意味着同样的推力只能推一半的距离。

### 2.3 地形数据结构建议

```typescript
interface MountainConfig {
  id: number;                // 0-3
  name: string;              // 显示名称
  height: number;            // 山顶高度（游戏单位）
  slopeAngle: number;        // 上坡坡度（度数或弧度）
  descentAngle: number;      // 下坡坡度（比上坡更陡）
  pushDistanceMultiplier: number;  // 推力系数
  flatTransitionLength: number;    // 山脚平地长度
  visualTheme: string;       // 视觉主题标识
  checkpoints: Checkpoint[]; // 检查点配置
  summitIngotReward: number; // 首次登顶锭奖励
}

interface Checkpoint {
  height: number;       // 触发高度
  goldReward: number;   // 金币奖励
}
```

## 3. 核心游戏流程变更

### 3.1 攀爬流程（改动）

```
玩家在商店选择起始山（默认=最高已解锁山）
    ↓
进入攀爬阶段，从该山的山脚（高度0）开始
    ↓
正常推石头，判定成功/失败逻辑不变
    ↓
┌─ 情况A：滑落到山脚（高度≤0）
│   → 结算本次攀爬金币 → 进入商店 → 重新选择山开始
│
├─ 情况B：体力耗尽
│   → 强制滑落到山脚 → 同情况A
│
└─ 情况C：到达山顶（高度 ≥ 山高度）
    → 触发登顶事件（见3.2）
    → 巨石自动滚下山的下坡面（玩家不操作，纯动画）
    → 经过平地，到达下一座山的山脚
    → 自动进入商店（体力重置为满）
    → 玩家买完东西后，从下一座山山脚开始攀爬
```

### 3.2 登顶事件

当玩家首次到达某座山的山顶时：

1. **一次性奖励**：发放该山的锭奖励（仅首次）
2. **解锁下一座山**：标记 `mountainsUnlocked[nextIndex] = true`
3. **更新默认起始山**：商店中默认选择变为下一座山

当玩家非首次到达山顶时：

1. 不发放锭奖励
2. 正常触发滚落动画和商店流程

### 3.3 失败滑落（改动）

滑落逻辑基本不变，但目标位置改为**当前山的山脚**（高度0），不是全局高度0。玩家在山2上滑落，只滑到山2的山脚，不会滚回山1。

## 4. 商店界面变更

### 4.1 新增：起始山选择器

在商店界面底部新增山脉选择组件：

```
┌─────────────────────────────────────┐
│  [升级列表...]                        │
│  [神器列表...]                        │
│                                     │
│  ─── 选择起始山 ───                   │
│  ○ 山1：塔耳塔洛斯丘陵 (2,000)        │
│  ● 山2：冥府斜坡 (5,000)    ← 默认    │
│  ○ 山3：奥林匹斯山腰 (12,000) [锁定]   │
│  ○ 山4：众神之巅 (24,000)    [锁定]    │
│                                     │
│  [ 再次攀爬 ]                         │
└─────────────────────────────────────┘
```

规则：

- 只显示已解锁的山可选，未解锁的灰显+锁定图标
- **默认选中**：玩家到达过的最高山（不是最高已解锁，而是当前应该爬的山）
- 玩家可以手动选回低级山（刷金币用）
- 选择状态保存在游戏状态中

### 4.2 商店触发时机

| 触发情况 | 行为 |
|---------|------|
| 滑落到山脚 | 结算金币 → 商店（默认=当前山） |
| 登顶后滚到下一座山脚 | 结算金币+锭 → 商店（默认=下一座山） |
| 游戏开始/重新开始 | 商店（默认=最高已解锁山） |

## 5. 登顶→下山→下一座山的动画流程

这是一段**纯表现**的过渡动画，玩家不操作：

```
1. 玩家推石头超过山顶高度
2. 播放登顶特效（一次性锭奖励弹出，如有）
3. 巨石越过山顶，开始沿下坡自动滚落
   - 摄像机跟随
   - 滚落速度逐渐加快（可复用现有滑落的加速逻辑）
4. 巨石经过山间平地
5. 巨石到达下一座山的山脚，停下
6. 画面淡出 → 进入商店
```

预估动画时长：2-4秒（可调）。

## 6. 地形渲染变更

### 6.1 现有逻辑（推测）

当前山坡是一个无限延伸的固定坡度斜面，摄像机跟随角色向上移动。

### 6.2 新逻辑

需要将地形改为**分段定义的折线/曲线**：

```
世界坐标X轴 → 水平距离
世界坐标Y轴 → 高度

地形函数 getTerrainHeight(worldX) → worldY
```

每座山的地形由以下控制点定义：

```
山脚起点 → 上坡（固定角度的直线或缓曲线）→ 山顶（圆弧过渡）→ 下坡（更陡的直线）→ 平地 → 下一座山脚
```

### 6.3 视觉区分

| 山脉 | 地面颜色/纹理建议 | 背景 |
|------|------------------|------|
| 山1 | 浅灰/浅棕 | 冥界入口，昏暗天空 |
| 山2 | 深灰/暗褐 | 更暗，雾气更浓 |
| 山3 | 深褐/暗红 | 火山岩质感，远处雷电 |
| 山4 | 近黑/深紫 | 众神领域，星空或极光 |

> 具体美术风格待定，先做颜色区分即可。

### 6.4 摄像机

摄像机逻辑基本不变（跟随角色，高度越高越远），但需要适配新的世界坐标系。角色不再只在Y轴移动，而是沿地形线移动（X和Y同时变化）。

## 7. 游戏状态变更

### 7.1 新增持久化状态

```typescript
interface GameState {
  // ... 现有字段 ...

  // 新增
  mountainsUnlocked: boolean[];    // [true, false, false, false]
  mountainsSummited: boolean[];    // [false, false, false, false] 用于锭奖励判定
  selectedMountainIndex: number;   // 商店中选择的起始山，默认0
  highestMountainReached: number;  // 到达过的最高山索引，用于默认选中
}
```

### 7.2 攀爬状态变更

```typescript
interface ClimbState {
  // ... 现有字段 ...

  // 新增
  currentMountainIndex: number;    // 当前攀爬的山
  currentMountainHeight: number;   // 当前山的高度上限
  pushDistanceMultiplier: number;  // 当前山的推力系数

  // 修改
  height: number;  // 现在是相对于当前山山脚的高度（0 = 山脚，mountainHeight = 山顶）
}
```

## 8. 开发优先级建议

| 优先级 | 任务 | 依赖 |
|--------|------|------|
| P0 | 地形数据结构 + 4山配置 | 无 |
| P0 | 攀爬逻辑适配（推力系数、相对高度、滑落到山脚） | 地形数据 |
| P0 | 登顶判定 + 解锁下一座山 + 锭奖励 | 攀爬逻辑 |
| P0 | 商店起始山选择器 | 解锁逻辑 |
| P1 | 地形渲染（分段坡度 + 山顶 + 下坡 + 平地） | 地形数据 |
| P1 | 登顶滚落过渡动画 | 地形渲染 |
| P1 | 每座山的视觉区分（颜色/背景） | 地形渲染 |
| P2 | 摄像机适配新世界坐标 | 地形渲染 |
| P2 | 山间过渡的音效/特效 | 过渡动画 |
